# 全国高校数据采集运行手册（Runbook）

日期：2026-03-02

## 1. 运行目标
- 全国范围持续更新院校、专业、录取、计划数据
- 在高考季保障高频增量
- 保证可追溯、可回滚、可审计

## 2. 任务分层与频次

### T1 基线任务（周更）
- 教育部院校名单同步
- 高校基础信息核对

### T2 核心录取任务（日更/高峰6-12小时）
- 各省考试院录取线/位次/批次
- 一分一段表

### T3 规则与专业任务（日更）
- 阳光高考与高校官网章程
- 专业选科与限制条件

### T4 画像增强任务（月更）
- 双一流/学科建设/就业统计

## 3. 调度策略
- 任务维度：`省份 x 来源 x 年份 x 数据类型`
- 并发策略：按来源限流，省份并行
- 重试策略：指数退避（1m, 5m, 30m），最多3次
- 失败转人工：连续失败>3次入告警

## 4. 标准化处理
- 统一批次命名（例：本科一批/本科批）
- 统一科类命名（文科/理科/物理类/历史类）
- 统一院校名称映射到 `university_uid`
- 保留原始文本字段便于追溯

## 5. 发布策略
1. 新数据进入 `staging`
2. 通过结构/范围/一致性校验
3. 冲突进入 `data_conflicts`
4. 通过审核后合并到 `production`
5. 发布后生成差异报告（新增/更新/删除）

## 6. 监控与报警
- 采集成功率 < 95% 报警
- 核心字段完整率 < 98% 报警
- 跨源一致率 < 95% 报警
- 高峰时延 > 12h 报警

## 7. 上线验收KPI（全国）
- 省份覆盖率：100%
- 院校匹配覆盖率（教育部名单对齐）：>= 99%
- 近5年核心录取字段覆盖率：>= 95%
- 数据一致性：>= 95%

## 8. 本周执行清单
1. 完成数据表扩展与迁移脚本
2. 接入来源登记表与运行日志
3. 先跑全国院校基线 + 省考试院核心字段
4. 搭建冲突池与人工审核页面（最小版）
5. 每日输出进度报表（覆盖率/完整率/一致性/时效）
