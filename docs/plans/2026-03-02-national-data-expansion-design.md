# 全国高校数据扩展方案（A+B+C）

日期：2026-03-02  
项目：高考志愿规划平台  
目标：全国覆盖 + 填报可用 + 高时效

## 1. 目标与范围

### 1.1 目标
1. **覆盖广度（A）**：覆盖全国省份、院校、核心招生维度；院校主档与教育部名单对齐。
2. **可用深度（B）**：支持志愿决策核心字段：分省录取线、位次、批次、科类/选科、专业计划。
3. **更新时效（C）**：在招生季实现高频增量更新，缩短“发布到入库”延迟。

### 1.2 范围
- 数据实体：院校、专业、招生章程、分省分年录取数据、分省计划、选科规则。
- 地域：全国（31个省级行政区域）。
- 时间：优先近 5 年，可回补历史年份。

---

## 2. 推荐实施模式：混合模式（官方源 + 自动采集 + 人工审核）

### 2.1 分层架构
- **L0 主数据层**：教育部高校名单、院校基础档案（唯一主档）
- **L1 规则层**：高校章程、专业目录、选科要求、学费学制
- **L2 结果层**：分省分年最低分、平均分、最低位次、批次、科类/选科
- **L3 特征层**：双一流/学科评估/就业及综合画像特征

### 2.2 采集流水线
`source -> fetch -> parse -> normalize -> validate -> merge -> publish`

每条数据保留：
- `source_type` / `source_url`
- `fetched_at`
- `parser_version`
- `content_hash`
- `confidence_score`

---

## 3. 数据源优先级

### P0（必须）
1. 教育部高校名单（院校主档基线）
2. 各省教育考试院（分省录取线、位次、计划、一分一段）

### P1（高价值补充）
3. 阳光高考（院校库/专业库/章程）
4. 各高校本科招生网（最新章程、专业选科/限制）

### P2（画像增强）
5. 双一流与学科建设公开名单
6. 统计年鉴等公开指标（仅做辅助，不替代录取事实）

---

## 4. 质量控制与风控

## 4.1 五道校验门
1. **结构校验**：必填、类型、取值范围（分数/位次）
2. **主键去重**：同省同年同批次同科类冲突检测
3. **跨源一致性**：省考试院与其他来源比对
4. **异常检测**：年份跳变、位次断层、批次映射错误
5. **人工复核**：冲突数据进入审核池，未复核不发布

## 4.2 数据SLA建议
- 可用性：核心查询接口 >= 99.9%
- 时效：高峰期关键数据发布后 6-12 小时入库
- 完整率：核心字段 >= 98%
- 一致性：跨源一致率 >= 95%

---

## 5. 分阶段执行计划

## Phase 1（1-2周）：全国底表与标准化
- 对齐教育部名单，建立 `university_uid` 与别名映射
- 统一批次/科类/选科标准枚举
- 建立抓取任务框架与失败重试机制

## Phase 2（2-4周）：录取核心可用
- 覆盖全国省考试院核心录取字段（近5年）
- 打通最低分、位次、批次、科类检索
- 完成冲突告警 + 审核池流程

## Phase 3（持续）：专业与画像增强
- 专业选科、学费学制、就业和学科特征增强
- 增量更新与回补自动化
- 监控与质量看板迭代
